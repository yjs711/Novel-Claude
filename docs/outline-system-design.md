## 一、五层大纲结构

```
系列 Bible (不变 — 世界观/境界系统/核心张力)
  │
  ├─ 卷大纲 (50万字/卷, 约10卷)
  │   每卷 = 独立起承转合 + 1个主线目标
  │
  ├─ 篇章 (10万字/篇, 约5篇/卷)
  │   每篇 = 1个完整剧情循环 + 可独立阅读
  │
  ├─ 章节 (3000字/章, 约33章/篇)
  │   每章 = 1个场景 + 1个情感基调 + 1个爽点
  │
  └─ 场景 Beat (800-1500字/场景)
      每场景 = 目标 → 障碍 → 反转 → 代价
```

## 三、循环引擎 (核心创新)

### 3.1 三循环模型
| 循环类型 | 跨度 | 功能 |
|----------|------|------|
| **微循环** | 3章 | 小爽点(打脸/突破/收获) |
| **中循环** | 10-15章 | 篇章级完整故事 |
| **大循环** | 50章/卷 | 换地图+境界突破 |

### 3.2 循环检测
生成篇章时自动验证:
- 是否在3章内完成一个微循环？
- 是否在15章内完成一个中循环(篇章)?
- 卷终是否有境界突破/地图切换?

## 四、情感弧线

每卷预设情感曲线:
```
卷1: 绝望 → 愤怒 → 拼搏 → 喜悦 (标准逆袭)
卷2: 喜悦 → 紧张 → 恐惧 → 震撼 (新地图碾压)
卷3: 孤独 → 温暖 → 愤怒 → 释放 (情感深度)
...
```

每章继承卷的情感位置，自动匹配情感参照。

### 6.2 补充「篇/弧」层级 (关键修正)

卷(50万字) 和 章(3000字) 之间插入弧线层:
```
卷 (50万字, 情感弧线)
  └─ 弧/篇 (10-15章, 完整剧情循环, 独立情感unit)
       └─ 章 (3000字, 场景+情感+爽点)
```

### 6.3 大纲回写循环

```
写作前: 大纲 → 意图编译(must-keep/must-avoid) → 写作
写作后: 审计偏离度 → 更新大纲状态 → 回流新事实
```

## 七、大师方法论修正 (关键)

### 7.1 新增: 素材研究阶段 (乌贼模式)
在生成大纲之前，必须先有素材积累:
```
阶段0: 素材研究 (新增!)
  ├─ 时代背景资料 (如: 维多利亚时代工人工资/工时)
  ├─ 饮食/服饰/建筑细节
  ├─ 民俗/信仰/社会结构
  └─ 生成《素材笔记.md》→ 大纲基于真实细节
```

### 7.2 大纲留白机制 (辰东模式)
```json
// 旧: 填满所有字段
{"chapter": 1, "title": "退婚", "scenes": [...]}

// 新: 只标关键节点, 其他留空让灵感填充
{"chapter": 1, "anchor": "退婚事件", "must_hit": ["羞辱","反击"],
 "free_zone": true, "scenes": []}  // free_zone=true 表示AI可自由发挥
```
比例: 20% 固定节点 + 80% 创作留白

### 7.3 循环驱动 (土豆模式)
不是"写完第1章写第2章", 而是:
```
微循环(3章) → 中循环(15章) → 大循环(50章换地图)
每个循环独立可读, 像游戏副本
```

### 7.4 伏笔随手扔 (乌贼模式)
- 写作时不预设伏笔目标
- 每章写完后, AI扫描文本标记"可回收点"
- 后期写作时检索: "有没有之前扔的伏笔可以用?"
- 乌贼原话: "前面随便埋的, 后面写的时候直接拿来用"
