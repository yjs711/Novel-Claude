# 超长篇大纲设计系统 — 500万字级网文架构

> 基于: 万订作者白蘸糖三级大纲 + DOME 动态层级生成 (2024) + LeStrTP 章级图 (2025) + Snowflake 雪花法

---

## 一、五层大纲结构

```
系列 Bible (不变 — 世界观/境界系统/核心张力)
  │
  ├─ 卷大纲 (50万字/卷, 约10卷)
  │   每卷 = 独立起承转合 + 1个主线目标
  │
  ├─ 篇章 (10万字/篇, 约5篇/卷)
  │   每篇 = 1个完整剧情循环 + 可独立阅读
  │
  ├─ 章节 (3000字/章, 约33章/篇)
  │   每章 = 1个场景 + 1个情感基调 + 1个爽点
  │
  └─ 场景 Beat (800-1500字/场景)
      每场景 = 目标 → 障碍 → 反转 → 代价
```

## 三、循环引擎 (核心创新)

### 3.1 三循环模型
| 循环类型 | 跨度 | 功能 |
|----------|------|------|
| **微循环** | 3章 | 小爽点(打脸/突破/收获) |
| **中循环** | 10-15章 | 篇章级完整故事 |
| **大循环** | 50章/卷 | 换地图+境界突破 |

### 3.2 循环检测
生成篇章时自动验证:
- 是否在3章内完成一个微循环？
- 是否在15章内完成一个中循环(篇章)?
- 卷终是否有境界突破/地图切换?

## 四、情感弧线

每卷预设情感曲线:
```
卷1: 绝望 → 愤怒 → 拼搏 → 喜悦 (标准逆袭)
卷2: 喜悦 → 紧张 → 恐惧 → 震撼 (新地图碾压)
卷3: 孤独 → 温暖 → 愤怒 → 释放 (情感深度)
...
```

每章继承卷的情感位置，自动匹配情感参照。

### 6.2 补充「篇/弧」层级 (关键修正)

卷(50万字) 和 章(3000字) 之间插入弧线层:
```
卷 (50万字, 情感弧线)
  └─ 弧/篇 (10-15章, 完整剧情循环, 独立情感unit)
       └─ 章 (3000字, 场景+情感+爽点)
```

### 6.3 大纲回写循环

```
写作前: 大纲 → 意图编译(must-keep/must-avoid) → 写作
写作后: 审计偏离度 → 更新大纲状态 → 回流新事实
```
